# FootballKitRadar Enterprise Architecture

## Product Surface

- `src/app`: Next.js App Router pages, SEO routes and API endpoints.
- `src/proxy.ts`: Next.js request proxy for baseline security headers and admin noindex rules.
- `src/components`: reusable premium commerce UI components.
- `src/data`: current demo/catalog configuration and source registry.
- `src/config/site.ts`: single source of truth for brand, canonical URL, repository and platform modes.
- `apps/mobile`: isolated Expo shell for the future React Native app. It has its own package manifest so Vercel does not install mobile-only dependencies for the web build.

## Commerce Intelligence

- `src/lib/ai-engine.ts`: seller trust, AI quality, smart deal score, image/OCR/logo signal simulation, price history analysis and product matching.
- `src/lib/affiliate.ts`: affiliate provider boundary, search normalization and safe redirect URL handling.
- `src/lib/images.ts`: image validation and fallback guarantees so products never render without imagery.
- `src/lib/rate-limit.ts`: in-memory local rate-limit scaffold; replace with Redis/Vercel Firewall for high traffic.

## Supabase Model

Core production tables live in `supabase/schema.sql`:

- Commerce: `products`, `offers`, `sellers`, `marketplaces`, `price_history`
- Intelligence: `product_matches`, `marketplace_trust_history`, `ai_scoring_events`, `ai_seller_signals`
- Growth: `seo_content_pages`, `seasonal_products`, `club_data`
- Monetization: `affiliate_clicks`, conversions and commission fields
- Users: `users`, `favorites`, `alerts`

## SEO

- Metadata and canonical URLs are centralized through `siteConfig`.
- `/sitemap.xml` and `/robots.txt` are generated by App Router metadata routes.
- Product pages emit `Product` JSON-LD.
- SEO collection pages emit `CollectionPage` JSON-LD.

## Scaling Path

1. Replace mock marketplace data with approved provider feeds.
2. Move rate limiting to Redis/Vercel Firewall.
3. Add scheduled ingestion jobs for price snapshots and seller drift.
4. Store image embeddings and OCR outputs in Supabase.
5. Promote SEO content from `seo_content_pages`.
6. Add observability for API latency, conversion tracking and Core Web Vitals.
